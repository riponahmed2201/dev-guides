# OSI Model (ওএসআই মডেল)

OSI (Open Systems Interconnection) হলো একটি কনসেপচুয়াল ফ্রেমওয়ার্ক যা নেটওয়ার্কের মাধ্যমে ডেটা কীভাবে এক ডিভাইস থেকে অন্য ডিভাইসে যায় তা ব্যাখ্যা করে। এতে মোট ৭টি লেয়ার থাকে।

---

## ১. OSI Model এর ৭টি লেয়ার

### লেয়ার ৭: Application Layer
ইউজার সরাসরি এই লেয়ারের সাথে ইন্টারেক্ট করে (যেমন- Chrome, Outlook)।
*Protocols: HTTP, HTTPS, FTP, DNS.*

### লেয়ার ৬: Presentation Layer
এটি ডেটাকে এমন ফরম্যাটে রূপান্তর করে যা অ্যাপ্লিকেশন লেয়ার বুঝতে পারে। এনক্রিপশন এবং কমপ্রেশন এখানে হয়।

### লেয়ার ৫: Session Layer
দুইটি ডিভাইসের মধ্যে কানেকশন তৈরি, ম্যানেজ এবং বন্ধ করা এর কাজ।

### লেয়ার ৪: Transport Layer
ডেটাকে ছোট ছোট সেগমেন্টে ভাগ করা এবং এরর-ফ্রি ডেলিভারি নিশ্চিত করা এর কাজ।
*Protocols: TCP, UDP.*

### লেয়ার ৩: Network Layer
এটি ডেটার জন্য সেরা রুট (Path) খুঁজে বের করে। এখানে আইপি অ্যাড্রেসিং কাজ করে।
*Devices: Router.*

### লেয়ার ২: Data Link Layer
ফিজিক্যাল লেয়ার দিয়ে ডেটা পাঠানোর আগে তাকে ফ্রেমে (Frame) ভাগ করা। এখানে ম্যাক (MAC) অ্যাড্রেস কাজ করে।
*Devices: Switch.*

### লেয়ার ১: Physical Layer
হার্ডওয়্যার লেভেল—বিট হিসেবে (০ এবং ১) ইলেকট্রিকাল সিগন্যাল বা অপটিক্যাল ফাইবারের মাধ্যমে ডেটা পাঠানো।

---

## ২. কেন এটি গুরুত্বপূর্ণ?
- নেটওয়ার্কের সমস্যা কোন লেয়ারে হচ্ছে তা দ্রুত খুঁজে পাওয়া যায় (Troubleshooting)।
- বিভিন্ন ভেন্ডরের হার্ডওয়্যার যেন একে অপরের সাথে কথা বলতে পারে তা নিশ্চিত করে।
